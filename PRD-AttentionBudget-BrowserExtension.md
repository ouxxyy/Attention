## 1. Problem Statement

### 1.1 用户痛点

知识工作者平均每天切换标签页/应用 **47次**，每次切换后需要 **23分钟** 才能完全恢复专注状态。这意味着：

- 每天约 **18小时** 的注意力被浪费在任务切换恢复上
- 用户知道自己分心，但缺乏**量化感知**和**柔性干预**机制
- 现有工具要么过于强硬（屏蔽网站引发对抗），要么过于被动（仅统计无干预）

### 1.2 证据

- **用户研究**: 深度访谈 8 位远程工作者，其中 6 位表示"知道该专注但控制不住"
- **行为数据**: 浏览器插件市场同类工具平均留存率 < 15%，用户抱怨"太粗暴"
- **竞品分析**: RescueTime 仅统计，StayFocusd 过于强制，Forest 专注移动端

### 1.3 我们解决什么

通过**极致量化 + 柔性引导**的组合，让用户真正"看见"自己的注意力消耗，在不引发抗拒的前提下主动优化行为。

  - [ ] 每次切换标签时，插件自动计算并显示消耗的注意力预算

## 5. Solution Overview

### 5.1 产品定位

**一句话价值**: 像会计记账一样追踪每一分钟注意力，用柔性引导而非强制屏蔽帮助你专注。

**核心差异化**:

1. **极致量化**: 每次切换 = 23分钟成本，让抽象的"分心"变成具体的数字
2. **柔性引导**: 不阻断、不惩罚，用心理机制（损失厌恶、即时反馈）引导行为
3. **完全隐私**: 数据仅存本地，无需账号，开箱即用

### 5.3 关键设计决策

| 决策     | 选择                 | 理由                     |
| -------- | -------------------- | ------------------------ |
| 数据存储 | 完全本地 (IndexedDB) | 隐私差异化 + 零运维成本  |
| 干预模式 | 可配置弹窗 (非强制)  | 尊重用户自主权，降低抗拒 |
| 成本计算 | 固定 23分钟/次切换   | 基于研究数据，简单可理解 |
| 日预算   | 480分钟 (8小时)      | 符合认知，可手动调整     |
| AI功能   | 本地模式，不上传数据 | 隐私优先原则             |

## 8. User Stories 详细规格

### Story 1: 实时注意力预算显示

**用户流程**:

1. 用户安装插件，首次使用时设置日注意力预算（默认480分钟）
2. 插件在浏览器右上角显示悬浮图标，显示剩余预算百分比
3. 每次切换标签页，预算自动扣减23分钟
4. 预算 < 20% 时图标变红，显示警告

**技术实现**:

- Background Service Worker 监听 `chrome.tabs.onActivated`
- 使用 `chrome.storage.local` 存储当前预算
- Badge API 更新图标数字显示

**边界情况**:

- 新标签页/Chrome内页不计入切换
- 同一域名内切换标签不计入
- 夜间模式自动刷新预算

### Story 2: 可配置干预弹窗

**用户流程**:

1. 用户在设置页配置"分心网站"列表（默认包含: twitter.com, reddit.com, youtube.com 等）
2. 当用户尝试访问分心网站时，弹窗拦截
3. 弹窗显示:
   - "这次切换将消耗 23 分钟注意力"
   - "今日剩余: X 分钟"
   - 按钮: [坚持当前任务] [仍然切换]
4. 用户可选择"记住我的选择(10分钟)"

**配置选项**:

- 拦截开关: 开/关
- 拦截模式: 所有分心网站 / 仅高优先级任务时
- 拦截时段: 可配置"专注时段"（如 9:00-12:00）
- 弹窗样式: 温和(可关闭) / 严格(需等待5秒)

### Story 3: 注意力流向报表

**日报内容**:

- 总专注时长 vs 分心时长
- 网站停留排行 (Top 10)
- 切换次数统计
- 注意力成本总计

**周报内容**:

- 7天趋势图
- 高效时段识别
- 注意力黑洞标注
- vs 上周对比

**数据隐私**:

- 所有数据本地存储
- 提供一键导出 JSON/CSV
- 提供一键清除所有数据

## 9. Appendix

### 9.1 竞品分析对比

| 竞品       | 核心模式   | 缺点           | 我们的优势          |
| ---------- | ---------- | -------------- | ------------------- |
| RescueTime | 后台统计   | 仅记录，无干预 | 实时预算 + 柔性引导 |
| StayFocusd | 强制屏蔽   | 粗暴，易绕过   | 可配置，尊重选择权  |
| Forest     | 移动端为主 | 浏览器支持弱   | 深度浏览器集成      |
| OneTab     | 标签管理   | 不解决分心根源 | 量化注意力成本      |

### 9.2 术语表

- **注意力预算**: 用户每日可用的专注时间总量（默认480分钟）
- **切换成本**: 每次任务切换导致的注意力损耗（固定23分钟）
- **分心网站**: 用户配置的会导致分心的网站列表
- **专注时段**: 用户配置的高优先级工作时间
